## 1. プロダクト概要

### 1.1 プロダクトの目的

**Rikai** は、AIを活用して学習ロードマップを自動生成・視覚化・進捗管理するSaaSプロダクトである。ユーザーが学びたいトピックを入力するだけで、個人最適化された学習経路をグラフィカルに提示し、学習の全体像把握と継続的な進捗追跡を支援する。

### 1.2 ターゲットユーザー

- 新しいスキルや技術を体系的に学びたい個人
- 学習計画を可視化・管理したいセルフラーナー
- チームの学習経路を設計・共有したいチームリーダー（将来拡張）

### 1.3 コアバリュー

| 価値 | 説明 |
|------|------|
| **自動生成** | トピック入力だけでAIが学習ステップを即座に構造化 |
| **視覚化** | ノード・エッジによるグラフィカルなロードマップ表示 |
| **進捗管理** | 各学習ステップの完了状況をリアルタイムで反映 |

## 2. 機能要件

### 2.1 ロードマップ生成機能

学習したいテーマをAIが分析し、段階的な学習経路を自動構築する機能。

#### 2.1.1 プロンプト入力

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーが学習したいトピックをテキストで入力する |
| **入力方法** | 単一テキストフィールド（自由記述） |
| **入力例** | `"機械学習を基礎から学びたい"` / `"Reactを使ったWebアプリ開発"` |

**要件詳細：**
- テキスト入力UIを提供し、ユーザーが学習テーマを自由記述できること
- 入力後、生成処理中はローディング表示を行うこと
- 入力値のバリデーション（空文字、極端に長いテキスト）を行うこと

#### 2.1.2 構造化データ生成

| 項目 | 内容 |
|------|------|
| **概要** | LLMを用いて学習ステップを階層・依存関係グラフとして生成する |
| **出力形式** | JSON（ノードリスト・エッジリスト） |
| **処理方式** | サーバーサイドでLLM APIを呼び出し、構造化データを返却 |

**JSON出力スキーマ（概要）：**

```json
{
  "title": "ロードマップタイトル",
  "nodes": [
    {
      "id": "node_1",
      "label": "学習項目名",
      "description": "詳細な解説テキスト",
      "level": 1
    }
  ],
  "edges": [
    {
      "source": "node_1",
      "target": "node_2"
    }
  ]
}
```

**要件詳細：**
- LLMのレスポンスは必ずバリデーションし、不正な構造の場合はリトライまたはエラーハンドリングを行うこと
- 生成ステップ数は学習テーマの複雑度に応じて自動調整されること（目安：5〜20ノード）

#### 2.1.3 初期描画

| 項目 | 内容 |
|------|------|
| **概要** | 生成されたJSONデータを元にグラフィカルなロードマップを描画する |
| **描画要素** | ノード（学習項目）、エッジ（順序・依存関係の線） |

**要件詳細：**
- 生成完了後、自動的にロードマップビューへ遷移すること
- ノードはラベル（学習項目名）を表示すること
- エッジは依存関係・学習順序を矢印で表現すること
- 初期レイアウトは自動計算により見やすく配置されること

### 2.2 ロードマップ閲覧・操作機能

生成されたロードマップをインタラクティブに閲覧・操作する機能。

#### 2.2.1 インタラクティブ・ビュー

| 操作 | 内容 |
|------|------|
| **ズーム** | マウスホイールまたはピンチ操作で拡大・縮小 |
| **パン（移動）** | ドラッグ操作でビューを移動 |
| **ノードドラッグ** | 個別ノードをドラッグして配置を変更 |

**要件詳細：**
- ズーム範囲は最小25%〜最大200%程度を目安とすること
- パン・ズーム操作は滑らかなアニメーションで表示すること
- ノードの配置変更（ドラッグ）も自動保存の対象とすること

#### 2.2.2 詳細表示

| 項目 | 内容 |
|------|------|
| **トリガー** | ノードのクリック・タップ |
| **表示内容** | 学習ステップ名、詳細解説テキスト、完了ステータス変更UI |
| **表示方法** | サイドパネルまたはモーダルダイアログ |

**要件詳細：**
- ノードクリック時、当該ノードの詳細情報をパネルに表示すること
- 詳細パネルはロードマップビューを隠さない形で表示されること（サイドパネル推奨）
- パネルの閉じるボタンまたは背景クリックで閉じられること

#### 2.2.3 進捗管理

| 項目 | 内容 |
|------|------|
| **ステータス種別** | 未着手 / 学習中 / 完了（3段階） |
| **操作方法** | 詳細パネル内のセレクト・ボタンで切り替え |
| **視覚的フィードバック** | ノードの色変化（例：未着手＝グレー、学習中＝イエロー、完了＝グリーン） |

**要件詳細：**
- ステータスはノードの色・スタイルに即座に反映されること
- 全体の完了率（例：`3/10 完了`）をロードマップ上部等に表示すること
- ステータス変更は自動保存でデータベースに即時反映されること
- DBのステータスカラムは `not_started` / `in_progress` / `completed` の3値で管理すること

### 2.3 ユーザー管理・保存機能

ユーザー認証およびデータ永続化に関する機能。

#### 2.3.1 認証基盤

| 項目 | 内容 |
|------|------|
| **認証ライブラリ** | Better Auth |
| **登録・ログイン方法** | メールアドレス＋パスワードのみ |
| **ソーシャルログイン** | 対応なし（将来拡張の余地あり） |
| **セッション管理** | Better Auth が提供するセッション管理機構を使用 |

**要件詳細：**
- パスワードはハッシュ化して保存すること（平文保存禁止）
- ログアウト機能を提供すること
- パスワードリセット機能は**スコープ外**（将来対応。メール送信基盤はResend等で別途整備予定）
- Better Auth のCloudflare Workers対応アダプターを使用すること

#### 2.3.2 データ永続化

| 保存対象 | 内容 |
|----------|------|
| **ロードマップ構造** | タイトル、ノードリスト、エッジリスト、作成日時 |
| **進捗ステータス** | 各ノードの完了・未着手状態 |
| **カスタム配置** | ユーザーがドラッグした際のノード座標（オプション） |

**要件詳細：**
- ロードマップは**自動保存**（操作後随時）で保存されること（明示的な保存ボタンは不要）
- 再ログイン・再訪問時に前回の状態が完全に復元されること
- データ削除機能（ロードマップ単位の削除）を提供すること

#### 2.3.3 マイページ

| 項目 | 内容 |
|------|------|
| **表示内容** | 作成済みロードマップの一覧（タイトル、作成日、進捗率） |
| **操作** | ロードマップの開く・削除 |

**要件詳細：**
- 一覧はカード形式またはリスト形式で表示すること
- 各カードに進捗率（例：`30%`）を視覚的に表示すること（プログレスバー等）
- 一覧は作成日時の降順でソートすること（デフォルト）

### 2.4 SaaS・決済機能

利用制限とStripeを用いたサブスクリプション管理機能。

#### 2.4.1 利用制限（ティア制）

| ティア | 内容 |
|--------|------|
| **無料プラン（Free）** | ロードマップ作成数：最大3件まで |
| **有料プラン（Pro）** | ロードマップ作成数：無制限 |

**要件詳細：**
- 無料ユーザーが作成数上限に達した場合、新規作成ボタンを無効化しアップグレード導線を表示すること
- 制限数は管理側で変更可能な設定値として管理すること（ハードコード禁止）
- 既存の作成済みロードマップはダウングレード後も閲覧・操作が可能であること

#### 2.4.2 サブスクリプション管理

| 項目 | 内容 |
|------|------|
| **決済基盤** | Stripe |
| **課金サイクル** | 月額サブスクリプション |
| **Webhook連携** | Stripeイベントを受信しDBの課金ステータスを同期 |

**要件詳細：**
- 決済フローはStripe Checkoutを使用すること
- 以下のStripe Webhookイベントを処理すること：
  - `checkout.session.completed`（サブスク開始）
  - `customer.subscription.deleted`（サブスク解約）
  - `invoice.payment_failed`（決済失敗）
- 課金ステータスはリアルタイムでDBに反映されること
- ユーザーがサブスクリプションをキャンセルできる管理ページを提供すること

## 3. 非機能要件

### 3.1 サーバーレス運用

| 項目 | 内容 |
|------|------|
| **目標** | インフラ管理コストの最小化 |
| **推奨構成** | Cloudflare Workers / Cloudflare Pages |
| **方針** | サーバーの手動管理・常時起動インスタンスを持たない構成 |

### 3.2 アクセシビリティ

| 項目 | 内容 |
|------|------|
| **準拠基準** | WAI-ARIA 1.1 準拠 |
| **対応内容** | キーボード操作、スクリーンリーダー対応、適切なARIAラベル付与 |
| **カラーコントラスト** | WCAG 2.1 AA 基準（コントラスト比4.5:1以上） |

### 3.3 レスポンシブ対応

| デバイス | 対応レベル |
|----------|-----------|
| **PCブラウザ** | メイン（フル機能） |
| **タブレット** | 主要機能を使用可能 |
| **モバイル（スマートフォン）** | 閲覧・進捗確認が可能（編集機能は制限あり可） |

### 3.4 パフォーマンス目標（参考値）

| 指標 | 目標値 |
|------|--------|
| **ロードマップ生成時間** | 30秒以内（LLM処理含む） |
| **初期ページロード（TTI）** | 3秒以内 |
| **ロードマップ再表示** | 1秒以内（DB読み込み） |

### 3.5 セキュリティ

- ユーザーは自身のデータにのみアクセスできること（横断アクセス禁止）
- APIキー・シークレットは環境変数で管理し、クライアントサイドに露出させないこと
- Stripe Webhookの署名検証を必ず実施すること

### 3.6 言語対応

| 項目 | 内容 |
|------|------|
| **対応言語** | 日本語のみ |
| **i18n対応** | 不要（将来拡張の余地あり） |
| **AIの出力言語** | Gemini APIへのプロンプトで日本語出力を指定すること |

## 4. システム構成・技術スタック

> ✅ 確定済みの技術選定を記載する。

| レイヤー | 採用技術 | 備考 |
|----------|---------|------|
| **フロントエンド** | Vite + React（SPA） | SSRなし・純粋なSPA構成 |

| **認証** | Better Auth | メール/PW認証、Cloudflare Workers対応 |
| **AI（LLM）** | Google Gemini API | 無料枠が大きいモデルを選定 |
| **決済** | Stripe | 月額サブスクリプション、Checkout + Webhook |
| **ホスティング** | Cloudflare Pages | フロントエンドの配信 |

## 5. 用語集

| 用語 | 説明 |
|------|------|
| **ロードマップ** | 学習項目（ノード）と順序（エッジ）で構成される学習経路の全体図 |
| **ノード** | ロードマップを構成する個々の学習ステップ・項目 |
| **エッジ** | ノード間の依存関係・学習順序を表す有向線 |
| **LLM** | Large Language Model。学習ロードマップの自動生成に使用するAIモデル |
| **Gemini** | GoogleのLLM。本プロダクトのロードマップ生成に使用するAPI |
| **React Flow** | Reactベースのグラフ描画ライブラリ。ノード・エッジの描画とインタラクションを担う |
| **Better Auth** | TypeScriptファーストの認証ライブラリ。Cloudflare Workers対応 |
| **Stripe** | 決済処理・サブスクリプション管理を行う外部決済サービス |
| **Webhook** | 外部サービス（Stripe等）からのイベント通知を受信するエンドポイント |
| **Cloudflare D1** | Cloudflare Workers上で動作するSQLiteベースのサーバーレスDB |
| **WAI-ARIA** | Webアクセシビリティのための技術仕様（W3C標準） |
| **TTI** | Time To Interactive。ページが操作可能になるまでの時間指標 |
| **TanStack Router** | React 向け型安全ルーター。ファイルベースルートと Vite プラグインでルートツリーを生成する |

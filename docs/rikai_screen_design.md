## 各画面の仕様

---

### 1. 新規登録 `signup`

**目的：** 新規ユーザーのアカウント作成

**フォーム構成：**

| フィールド | バリデーション |
|------------|---------------|
| メールアドレス | 必須・メール形式 |
| パスワード | 必須・8文字以上 |
| パスワード確認 | 必須・パスワードと一致 |

**主要アクション：**
- 「登録する」ボタン → Better Auth でアカウント作成 → `/dashboard` へ遷移
- 「ログインはこちら」→ `/login`

**エラー表示：**
- メール重複・バリデーションエラーはフィールド直下にインライン表示

---

### 2. ログイン `login`

**目的：** 既存ユーザーの認証

**フォーム構成：**

| フィールド | バリデーション |
|------------|---------------|
| メールアドレス | 必須・メール形式 |
| パスワード | 必須 |

**主要アクション：**
- 「ログイン」ボタン → 認証成功後 `/dashboard` へ遷移
- 「アカウント登録はこちら」→ `/signup`

**エラー表示：**
- 認証失敗時は「メールアドレスまたはパスワードが正しくありません」をフォーム上部に表示（セキュリティのため詳細を出さない）

---

### 3. ダッシュボード `dashboard`

**目的：** 作成済みロードマップの管理と新規作成への導線

**現状：** URL は **`/`**。ページ見出しは「ロードマップ」。カードの進捗はバッジ（％完了）のみで、要件のプログレスバー行は簡略化されている場合あり。新規作成は**モーダル**（`/generate` 相当の独立画面は未実装）。

**レイアウト構成：**

```
┌─────────────────────────────────────┐
│ ヘッダー（ロゴ / ナビ / ユーザーメニュー）│
├─────────────────────────────────────┤
│ ページタイトル「マイロードマップ」          │
│ [＋ 新規作成] ボタン                    │
├─────────────────────────────────────┤
│ ロードマップカード一覧（グリッド）          │
│ ┌──────────┐ ┌──────────┐           │
│ │ タイトル   │ │ タイトル   │           │
│ │ 作成日    │ │ 作成日    │           │
│ │ 進捗バー  │ │ 進捗バー  │           │
│ │ [開く][削除]│ │ [開く][削除]│          │
│ └──────────┘ └──────────┘           │
└─────────────────────────────────────┘
```

**カード表示項目：**
- ロードマップタイトル
- 作成日時
- 進捗率（プログレスバー＋ `完了数/総数` テキスト）
- 「開く」ボタン → `/roadmap/:id`
- 「削除」ボタン → 確認ダイアログ → 削除実行

**無料プラン上限時の挙動：**
- 3件達成時、「＋ 新規作成」ボタンを非活性化
- バナーで「Proプランにアップグレードすると無制限に作成できます」を表示 → `/pricing` リンク

**空状態（0件）：**
- 「まだロードマップがありません。最初のロードマップを作成しましょう！」のイラスト＋テキスト＋作成ボタン

---

### 4. ロードマップ生成 `generate`

**目的：** トピック入力からAIによるロードマップ生成の実行

**レイアウト構成：**

```
┌─────────────────────────────────────┐
│ ヘッダー                              │
├─────────────────────────────────────┤
│        「何を学びますか？」              │
│  ┌──────────────────────────────┐   │
│  │ テキスト入力フィールド（大）      │   │
│  └──────────────────────────────┘   │
│  入力例：「機械学習を基礎から」「React入門」│
│        [ロードマップを生成する]          │
└─────────────────────────────────────┘
```

**生成中の状態：**
- ボタンをローディングスピナーに切り替え
- 「AIがロードマップを作成しています...」のメッセージ表示
- 入力フィールドを非活性化

**生成完了後：**
- 作成されたロードマップの `/roadmap/:id` へ自動遷移

**現状（モック）：** 一覧上のモーダルから作成。完了後はモーダルを閉じて一覧に追加するのみ（**詳細 URL への自動遷移は未実装**）。

**エラー時：**
- 「生成に失敗しました。もう一度お試しください。」をフォーム上部に表示
- 入力内容は保持したまま再試行可能

**バリデーション：**
- 空文字：送信不可（ボタン非活性）
- 文字数上限：200文字

### 5. ロードマップビュー `roadmap`

**目的：** 生成されたロードマップの閲覧・操作・進捗管理のメイン画面

**レイアウト構成：**

```
┌───────────────────────────────────────────┐
│ ヘッダー（ロゴ / ロードマップタイトル / メニュー）│
├─────────────────────┬─────────────────────┤
│                     │                     │
│   グラフエリア         │   詳細サイドパネル      │
│   （React Flow）     │   （ノード選択時に表示）  │
│                     │                     │
│   ズーム/パン/        │   - ノード名          │
│   ノードドラッグ対応   │   - 説明テキスト       │
│                     │   - ステータス切替UI   │
│                     │                     │
├─────────────────────┴─────────────────────┤
│ フッターバー：進捗サマリー（完了 3 / 10）・操作ヒント│
└───────────────────────────────────────────┘
```

**グラフエリアの仕様：**

| 要素 | 仕様 |
|------|------|
| ノード | 学習項目名を表示。ステータスに応じて色変化 |
| ノード色 | 未着手＝グレー、学習中＝イエロー、完了＝グリーン |
| エッジ | 有向矢印で学習順序を表現 |
| ズーム | マウスホイール / ピンチ（25%〜200%） |
| パン | 背景ドラッグで移動 |
| ノードドラッグ | 個別ノードの位置を変更可能（自動保存） |
| ミニマップ | 右下に全体俯瞰用のミニマップを表示 |
| フィットボタン | 全ノードが画面内に収まるようリセット |

**詳細サイドパネルの仕様：**

| 要素 | 仕様 |
|------|------|
| 表示トリガー | ノードクリック |
| ノード名 | 大きめのテキストで表示 |
| 説明テキスト | AIが生成した学習ステップの詳細説明 |
| ステータス切替 | 「未着手」「学習中」「完了」の3ボタンまたはセレクト |
| 閉じる | ✕ボタン or パネル外クリック |

**ヘッダーメニュー：**
- 「← ダッシュボードに戻る」
- 自動保存インジケーター（「保存済み」「保存中...」）

**現状（モック）：** 一覧相当画面（`/`）へ戻るテキストリンク（「ロードマップ一覧に戻る」）。**サイドパネルなし**（ノードカード内でラベル・説明・ステータスバッジ・トグル）。**ミニマップ・フッターバー進捗サマリーなし**（ヘッダー右に進捗率％と細いプログレスバーで代替）。ノード位置の永続化なし。

### 6. 料金プラン `pricing`

**目的：** Free / Pro プランの比較とStripe決済への導線

**レイアウト構成：**

```
┌─────────────────────────────────────┐
│        「シンプルな料金プラン」          │
├──────────────┬──────────────────────┤
│   Free       │   Pro                │
│   ¥0/月      │   ¥980/月            │
│              │                      │
│ ・作成3件まで  │ ・作成無制限           │
│ ・全機能利用可 │ ・全機能利用可          │
│              │ ・優先サポート（将来）   │
│ [現在のプラン] │ [Proにアップグレード]   │
└──────────────┴──────────────────────┘
```

**主要アクション：**
- 「Proにアップグレード」→ Stripe Checkout へリダイレクト
- ログイン済みかつProユーザーは「現在のプラン」をどちらにも表示
- 未ログインユーザーは「Proにアップグレード」クリック → `/login` へリダイレクト後、決済へ

---

### 7. アカウント設定 `settings`

**目的：** ユーザー情報の確認とサブスクリプション管理

**セクション構成：**

**① アカウント情報**

| 項目 | 内容 |
|------|------|
| メールアドレス | 表示のみ（変更は将来対応） |
| 登録日 | 表示のみ |

**② プラン・サブスクリプション**

| 状態 | 表示内容 |
|------|---------|
| Freeユーザー | 現在のプラン表示 ＋「Proにアップグレード」ボタン → `/pricing` |
| Proユーザー | 現在のプラン・次回更新日 ＋「サブスクリプションをキャンセル」ボタン |
| キャンセル済み | 有効期限日 ＋「再開する」ボタン |

**③ ログアウト**
- 「ログアウト」ボタン → セッション破棄 → `/login` へリダイレクト

## 画面遷移フロー

### 要件上の遷移（認証後を想定）

> アプリのエントリーポイントは `/login`。LPからの流入は Next.js 側で `/signup` または `/login` へ誘導する。

```
[login] ←→ [signup]
    │
    └─→ [dashboard]
              │
              ├─→ [generate] ──→ [roadmap/:id]
              │
              ├─→ [roadmap/:id]
              │
              ├─→ [pricing]
              │
              └─→ [settings]
```

### 現状の遷移（認証なしモック）

```
[/]  ──→  [/roadmap/:id]
  │              │
  │              └──→ [/]（戻るリンク）
  │
  ├──→ モーダル（作成）
  │
  └──→ [/settings]
```

**リダイレクトルール（要件）：**

| 条件 | 挙動 |
|------|------|
| 未ログインで認証必要画面にアクセス | `/login` へリダイレクト |
| ログイン済みで `/login`, `/signup` にアクセス | `/dashboard` へリダイレクト |
| 存在しないロードマップIDにアクセス | `/dashboard` へリダイレクト |

**現状：** 上記リダイレクトは**未適用**。存在しない ID は `/roadmap/$roadmapId` 上でメッセージ＋一覧へのリンクを表示。
